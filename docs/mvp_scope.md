# MVP Scope

## MVPの目的
最初のMVPでは、IDMLやPDFの難しい処理に入る前に「このエディタで書くと気持ちいい」と感じられるかを検証する。

## 必須機能
1. 爆速起動・即執筆
2. テキストエディタ＋チャットウィンドウ
3. 草稿モード / 編集モード
4. 段落文字数表示
5. 仮想行幅プレビュー（30W×4Lなど）
6. リアルタイム軽量ハイライト
7. 50文字以降から始まる遅延型解析
8. 超高耐久保存
9. 文字サイズ・行間・テーマ切替
10. タブまたは最近使った原稿への高速復帰

## MVPで実装しない機能
- IDML本文編集
- PDF注釈書き出し
- Web事実確認
- 企画書参照チェック
- 画像キャプション管理
- 記者ハンドブック相当校正
- 課金管理

## 重要な非機能要件
- 入力を絶対に止めない
- 保存で待たせない
- AI処理で本文入力を重くしない
- 起動時に重い処理をしない
- 元ファイルを直接壊さない
