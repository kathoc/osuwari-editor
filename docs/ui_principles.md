# UI Principles

## 基本レイアウト
- 左：テキストエディタ
- 右：チャット / 提案 / 補足情報
- 下部または右下：段落文字数、保存状態、AI状態

## 最重要体験
起動した瞬間に書けること。AIが準備中でも、本文入力を止めない。

## ハイライト
- 入力開始直後は表示しない
- 50文字以降から軽く表示
- 過去に書いた部分から徐々に濃くする
- 赤波線だらけにしない
- 草稿モードでは緩く、編集モードでは厳しく

## ズーム
- 文字サイズは常時変更できる
- 長時間使っていたサイズへスライダーが吸着する
- ルビや字形確認の一時拡大から戻りやすくする

## 相棒演出
- 本文領域には入れない
- 常時キャラ化しない
- バックグラウンド処理中に画面端で小さく反応する程度
- 最初のMVPでは未実装でもよい
