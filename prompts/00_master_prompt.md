# Master Prompt for Codex / Claude Code

あなたは、AI支援型の原稿執筆エディタを試作するシニア開発者です。

目的は、爆速で気持ちよく原稿を書けるローカルWebエディタを作ることです。AI機能は重要ですが、最初のMVPでは「起動の速さ」「入力の軽さ」「保存の堅牢性」「段落単位の編集体験」を最優先してください。

## 必ず守る思想
- アプリ起動後、最初の1文字を最速で入力できること。
- AI、PDF、IDML、画像、事実確認などの重い処理は遅延ロードすること。
- 入力中にAI処理でカーソルや入力を止めないこと。
- 本文を勝手にAIで書き換えないこと。草稿モード以外は承認制にすること。
- 原稿消失を極限まで避けること。通常のAutosaveではなく、差分ログ保存の思想を持つこと。
- 専用Vaultやプロジェクト作成を強制しないこと。

## 技術方針
- React + TypeScript + Vite
- Node.js backend
- SQLite
- まずAIはモックでよい
- 将来CLI AIアダプタを差し替えられる構造にする

## 最初に作るもの
1. Vite React アプリ
2. Node.js サーバー
3. エディタ画面
4. チャット欄
5. 段落文字数表示
6. 仮想行幅プレビュー（30W×4L）
7. 50文字以降からの遅延型ハイライト
8. 差分ログ保存の土台
9. 文字サイズ・行間・ライト/ダーク/システムテーマ
10. 機能オンデマンド読込の骨組み

## 実装前にやること
まず `docs/implementation_plan.md` を作成し、MVPの実装順序、ファイル構成、データモデル、未実装機能のスタブ方針を書いてください。

その後、最小構成で動くプロトタイプを実装してください。
